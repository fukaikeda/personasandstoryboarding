Evaluating the 3DP User Experience

In this page, we will be looking at the user interface of the 3D Printers in the Brown Design Workshop, a student makerspace.

Background Information

The Brown Design Workshop is a makerspace located in Brown University's engineering building, open to all. It's ususally busy with students and community members realazing their creative ambitions.

Although it is a very exciting space, it has over 1000 members and this means the place is often very busy and welcomes a variety of users. Sometimes, as a monitor, I notice users have issues in the interaction with different components.

Therefore, I decided to look into the interaction and evaluating the users needs, to help us find a long-term solution for increasing usability.

The Prusa Interface

The Prusas are the printers that the BDW uses in its space. While it has a relatively simple interface, with one SD card slot, a knob to twist and press, and another cancel button, this simpleness may limit the controls possible on the interface. The LCD screen is also very small, and does not respond to input, which limits the information that users can both recieve and input.

One concern I had after interaction with the interface itself is how many of the settings are 'bundled' together, and the difficult callibration settings are often mixed up with the print settings, which may provide a 'black hole' experience for the beginning users.

Recording Observations

Based on the initial discoveries I had with the Prusa printer itself, I decided that my observations were going to be of three different users: a beginning, intermediate, and advanced users. By observing the actions of each level of user, I could get insight into a variety of experiences.

User A (Beginner)
- User had trouble getting started, not sure how to navigate the space
- User didn't remove any prints and waited around
- User spent time on the computer interface
- User asked another user if printer was open
- User asked monitor to check their settings


User B (Intermediate)
- User got plates off of open printers to use them
- User had prints sliced and ready, and just uploaded them
- User switched filament to their own
- User seemed to want another printer, and checked waiting times on all the printers


User C (Expert)
- User removed done prints
- User stopped failed prints and callibrated them
- User prints a print as test for fixed printer
- User removes personal filament from printer


I also asked questions to users to better understand their interactions with the 3d printers.


1. What do you find most challenging when using 3DP?
  • User A (Beginner): Understanding settings and figuring if I'm doing things correctly
  • User B (Intermediate): Trying to deal with prints that fail, finding an open printer
  • User C (Expert): Fixing the 3DPs due to misuse.

2. What do you enjoy most about 3DP?
  • User A (Beginner): The opportunity to make anything
  • User B (Intermediate): Ideating really quickly! I can prototype so easily
  • User C (Expert): Having them as a useful and free resource here at Brown

3. What improvements do you want to see on the 3DP?
  • User A (Beginner): Resources to help people like me print fasters
  • User B (Intermediate): Some kind of system to book printers so I don't have to wait
  • User C (Expert): More users who respect the rules and don't break them

From these interviews, we find that each level user has different frusterations with the interaction of a 3d printer. It is observed that beginning users have more issues with the software-specific interaction components of the 3d printer experience, while more advanced users have trouble with the systematic interactions within the experience, that is more related to interpersonal expereience. This may mean that the 3dp interface is quite easily learnable, but not the best callibrated towards the nature of the space.

Personas

As mentioned in the last section, these users have such different experiences. Therefore, I begin to focus on a more niche user. I do this by creating two personas based on the observations I conducted. They are both beginners, but have different struggles.

Stephen is tech-savvy, but he has trouble with asking for help. His biggest interface issue is troubleshooting. Users like him are common, as many people traditionally have it down but struggle in uncommon situations.

Tiffany has no trouble asking for help, but the software is difficult for her to use. Her biggest challenge is the software interface and settings. Users like her are common too, as many people do not use the 3dp software often.

Storyboarding

Then, I focus even more on a specific user by taking their journey and storyboarding it. This process allows us to understand the specific points in a user's journey and their emotions in each, highlighting key points of improvement.


From this activity, we can understand that the lack of support and confusion is a key point in a negative experience of a beginning user, and that it can scare away users from using the devices. This is both something that the software can improve on (beginner-friendly debugging) but also the environment itself (support system).

Reflections

This activity helped me understand the different users of the 3D printers in the BDW and how each users interacts with them. Particularly, reviewing the users that are in a category the 'furthest away' from us helped me realize what changes are needed to make the technology useful for everyone.

While software changes are difficult to implement on my level, there are a variety of methods for myself to be involved in increasing the usability, such as providing more directed education during workshops, creating signage to help users, and be avaliable to answer questions. I want to implement these changes to make the 3DP experience better for a wide audience.
